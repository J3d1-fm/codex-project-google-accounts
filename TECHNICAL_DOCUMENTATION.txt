Codex Project Google Accounts technical documentation
Version: v0.1.3
Date: 2026-05-13

Purpose
-------
This repository packages a local Codex MCP plugin that lets Codex route Gmail, Google Calendar, and Google Drive work through a Google account bound to a local project folder.

Architecture
------------
1. Codex loads .codex-plugin/plugin.json.
2. The plugin declares .mcp.json as its MCP server config.
3. The MCP server runs bin/server.mjs with Node.js.
4. Project bindings are stored outside the repository under:
   ~/.codex/project-google-accounts/projects.json
5. OAuth tokens are stored outside the repository under:
   ~/.codex/project-google-accounts/tokens/
6. A project-local hint can be stored under:
   <project>/.codex/google-account.json

Fast-path rule
--------------
When a project binding is already known, the agent should call the requested Gmail, Calendar, or Drive tool directly. Profile, status, reconnect, and account diagnostics are fallback steps only when the binding is unknown, the user asks for diagnostics, or a direct operation fails.

Attachment support
------------------
The Gmail tools `gmail.send`, `gmail.create_draft`, and `gmail.reply_thread_latest` accept an optional `attachments` array.

Each attachment can use:
- path: local file path to read and attach
- filename: optional output filename
- mime_type: optional MIME type
- content_base64: optional inline base64 content when path is not supplied

Validation
----------
Run:
node --check bin/server.mjs

Run:
python3 /path/to/quick_validate.py skills/project-google-accounts

Security checklist
------------------
Before publishing, verify that the repository does not contain:
- access tokens
- refresh tokens
- OAuth client secrets
- personal email addresses
- personal local filesystem paths
- logs
- project-specific private documentation

Rollback
--------
Use git tags/releases for rollback. Gmail attachment support starts in v0.1.3.
